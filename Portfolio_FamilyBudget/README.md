# Портфолио: Проект «Семейный бюджет»

**Роль:** Project Manager (учебный проект, Школа 21)  
**Период:** 4 месяца 
**Команда:** 4 человека
**Инструменты:** Excel, Figma, Kaiten, Confluence-стиль

---

## 📁 Структура репозитория

| `01_Requirements_UserStories.pdf` | Бизнес/пользовательские/функциональные требования + 6 User Stories с DoD |
| `02_Budget_Full.xlsx` | Полная смета проекта (~3.76 млн руб, 10 ролей, задачи по этапам) |
| `02_Budget_Screenshot.pdf` | Скриншот таблицы полной сметы проекта (~3.76 млн руб, 10 ролей, задачи по этапам)|
| `03_TaskBoard_Screenshot.pdf` | Kanban-доска: Бэклог → Очередь → В работе → Тест → Готово |
| `04_Prototype_StatusUpdate.pdf` | Low-fi прототип сценария «Приглашение участника» (7 экранов) + статусный апдейт |
| `05_Risk_Register.pdf` | Реестр рисков: 5 рисков с оценкой вероятности, влияния и планом митигации |
| `06_Deviation_Analysis.pdf` | Анализ отклонений: сроки, качество, бюджет/комплаенс + коммуникация |
| `07_Final_Presentation.pdf` | Презентация для стейкхолдеров (8 слайдов) |
| `08_Handover_Plan.pdf` | План передачи проекта в поддержку (SLA, инциденты, документация) |

## 🎯 Ключевые результаты

- **Бюджет:** ~3.76 млн руб на 4 месяца (10 ролей, почасовые ставки 1,800–3,200 руб/час)
- **Прототип:** 7 экранов low-fi wireframes, 4 точки отказа с решениями
- **Риски:** 5 выявленных рисков (технические, юридические, ресурсные) с планом митигации
- **Методология:** Agile/Scrum, 2-недельные спринты, MVP из 4 User Stories
- **Передача в поддержку:** готовый план с SLA, инструкциями и ответственными

---

## 🛠️ Навыки, продемонстрированные в проекте

- Сбор и декомпозиция требований (бизнес/пользовательские/функциональные/нефункциональные)
- Оценка трудозатрат и бюджетирование в Excel
- Прототипирование пользовательских сценариев (Figma, low-fi)
- Управление рисками: идентификация, оценка, митигация
- Фасилитация встреч и коммуникация с распределённой командой
- Подготовка документации для передачи продукта в эксплуатацию
